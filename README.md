## Project Overview
Análisis integral de una prueba A/B diseñada para evaluar el impacto de un nuevo sistema de recomendaciones en el embudo de conversión de un e-commerce. A través de la evaluación de los datos, el proyecto identifica no solo el rendimiento de la nueva funcionalidad, sino también anomalías críticas en la captura de muestras (Sample Ratio Mismatch - SRM) y el ruido estadístico generado por campañas de marketing navideño.

## 🛠 Tech Stack
* **Language:** Python
* **Libraries:** Pandas, Plotly, SciPy / Statsmodels (para pruebas estadísticas)
* **Environment:** VS Code / Jupyter Notebook (Linux)

## 📈 Key Process
1. **Limpieza y Preprocesamiento:** Ajuste de los datasets del experimento y registros de eventos de usuarios.
2. **Análisis del Embudo de Conversión:** Mapeo de las etapas del usuario mediante gráficos de embudo (*funnel plots*) interactivos con Plotly para visualizar caídas y cuellos de botella.
3. **Comprobación de Validez:** Detección de un desequilibrio de muestras (SRM) en la división de tráfico entre los grupos A y B.
4. **Análisis de Estacionalidad:** Aislamiento del ruido generado por eventos de tráfico atípico (marketing de Navidad y Black Friday).
5. **Pruebas de Hipótesis:** Evaluación de la significancia estadística en las métricas clave de conversión.

---

## 💡 Conclusions
* El nuevo sistema de recomendaciones **no mejoró** la conversión final de manera significativa o positiva.
* Se detectó que el **Grupo B sufre una pérdida masiva de usuarios** ("churn") justo después de la etapa de login, lo que distorsiona severamente la parte alta del embudo.
* **Veredicto Técnico:** La prueba es **inválida**. Debido a las interferencias externas (marketing navideño) y a los graves problemas de implementación técnica (SRM), los resultados actuales no son confiables para la toma de decisiones empresariales.

## 🚀 Recommendations
1. **Detener la Implementación:** No se recomienda pasar el nuevo sistema de recomendaciones a producción, ya que los datos sugieren que actualmente degrada la experiencia del usuario o no genera el impacto esperado.
2. **Investigación Técnica Inmediata:** Auditar el sistema de división de tráfico para encontrar y corregir el bug que causó el desequilibrio de las muestras antes de planear un nuevo test.
3. **Análisis del Churn en el Grupo B:** Investigar el origen de la fuga de usuarios post-login (revisar tiempos de carga, bugs de interfaz o fricciones introducidas en el nuevo embudo).
4. **Repetición del Experimento:** Una vez solucionados los problemas técnicos, relanzar la prueba A/B en un periodo "limpio" y sin estacionalidades fuertes para poder medir el verdadero efecto del sistema.

# 🎄 A/B Testing: When SRM and Seasonality Break the Funnel

## Project Overview
A comprehensive analysis of an A/B test designed to evaluate a new recommendation system's impact on an e-commerce conversion funnel. Beyond just measuring conversion rates, this project successfully identifies critical anomalies in data collection (Sample Ratio Mismatch - SRM) and statistical noise generated by Christmas marketing campaigns, ultimately preventing a flawed deployment.

## 🛠 Tech Stack
* **Language:** Python
* **Libraries:** Pandas, Plotly, SciPy / Statsmodels
* **Environment:** VS Code / Jupyter Notebook (Linux)

## 💡 Conclusions
* The new recommendation system **did not significantly or positively improve** the final conversion rate.
* **Group B suffers a massive user drop-off** (churn) immediately after the login stage, severely distorting the upper funnel.
* **Technical Verdict:** The test is **invalid**. Due to external interference (holiday marketing) and critical implementation bugs (SRM), the current results are entirely unreliable for business decision-making.

## 🚀 Recommendations
1. **Halt Implementation:** Do not deploy the new recommendation system into production. Current data suggests it degrades the user experience and fails to deliver the expected impact.
2. **Technical Investigation:** Audit the traffic-splitting system immediately to fix the underlying bug causing the sample imbalance (SRM).
3. **Analyze Group B Churn:** Investigate the massive post-login drop-off in Group B to rule out loading issues or UI friction specific to the new funnel.
4. **Repeat the Experiment:** Once the bugs are fixed, relaunch the A/B test during a "clean" period (avoiding Black Friday/Christmas anomalies) to properly isolate and measure the system's true effect.
